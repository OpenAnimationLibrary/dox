\doxysubsection{Detailed Description}
\mbox{\hyperlink{class_stage_1_1_visitor}{Stage\+::\+Visitor}} is the abstract class model related to {\itshape  scene visiting \texorpdfstring{$<$}{<}\textbackslash{}I\texorpdfstring{$>$}{>}.}

{\itshape  Iterating a scene to perform some task on its content is a typical task in Toonz. Scene visiting is here intended as an implementation of the {\itshape  visitor-\/visitable \texorpdfstring{$<$}{<}\textbackslash{}I\texorpdfstring{$>$}{>} pattern applied to Toonz scenes, where the {\itshape visitable} objects are of type \mbox{\hyperlink{class_stage_1_1_player}{Stage\+::\+Player}}, and visitors are reimplemented from the \mbox{\hyperlink{class_stage_1_1_visitor}{Stage\+::\+Visitor}} class. ~\newline
~\newline
 The \mbox{\hyperlink{class_stage_1_1_player}{Stage\+::\+Player}} class is the rough equivalent of a level at a specific scene frame (the term {\itshape \mbox{\hyperlink{class_stage_1_1_player}{Player}}} being more legacy rather than having a definite sense I guess), which in particular can be seen as an {\itshape image} occurring in scene compositing. \mbox{\hyperlink{class_stage_1_1_player}{Stage\+::\+Player}} actually have a lot more insights about onion skinning, image placement, and other stuff -\/ be sure to check it thorough. ~\newline
~\newline
 When visiting a scene, the Players are supplied to \mbox{\hyperlink{class_stage_1_1_visitor}{Stage\+::\+Visitor}} instances sorted in compositing order -\/ so e.\+g. the first \mbox{\hyperlink{class_stage_1_1_player}{Player}} is intended {\itshape below} the last one.}}
