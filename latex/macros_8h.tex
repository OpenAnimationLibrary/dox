The TCG\+\_\+\+ASSERT instruction is an assertion that also provides a valid failure statement in case the asserted expression fails. 

Assert statements are used to enforce pre-\/ and post-\/conditions to algorithms, in a way that is {\bfseries{not}} intended to be broken -\/ in other words, after a failed assert statement we should expect nothing more than undefined behavior. The assert statement itself will signal a failed expression, {\itshape  and then will force the program to quit \texorpdfstring{$<$}{<}\textbackslash{}I\texorpdfstring{$>$}{>}.}

{\itshape  However, there exist cases in which at the same time\+:}

{\itshape  \begin{DoxyItemize}
\item It is {\itshape known} that a pre/post-\/condition could be possibly broken \item It should still be considered an error \item Yet, a worst-\/case bail-\/out policy must be provided\end{DoxyItemize}
When this happen, you should\+:}

{\itshape  \begin{DoxyItemize}
\item Feel offended -\/ it should {\bfseries{really not\texorpdfstring{$<$}{<}\textbackslash{}B\texorpdfstring{$>$}{>} happen. Errors in meeting requirements should never be tolerated }}\item {\bfseries{ Use TCG\+\_\+\+ASSERT. It does not solve things, but is a simple statement and defines a trackable (grep-\/able) string in the project }}\end{DoxyItemize}
