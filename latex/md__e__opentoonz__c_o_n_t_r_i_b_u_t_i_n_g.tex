This document describes some points about the contribution process for Open\+Toonz.\hypertarget{md__e__opentoonz__c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md1}{}\doxysection{Pull-\/requests}\label{md__e__opentoonz__c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md1}
The Open\+Toonz organization loves any kind of contributions, such as fixing typos and code refactoring. If you fixed or added something useful to Open\+Toonz, please send pull-\/requests to us. We will first review the request, then we will accept it, add comments for rework, or decline it.\hypertarget{md__e__opentoonz__c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md2}{}\doxysubsection{Workflow}\label{md__e__opentoonz__c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md2}
0. {\ttfamily fork} Open\+Toonz to your Git\+Hub account from {\ttfamily opentoonz/opentoonz}.
\begin{DoxyItemize}
\item (use the {\ttfamily fork} button at the \href{https://github.com/opentoonz/opentoonz}{\texttt{ https\+://github.\+com/opentoonz/opentoonz}}) 0. {\ttfamily clone} the repository.
\item {\ttfamily git clone git@github.\+com\+:your-\/github-\/account/opentoonz.\+git}
\item {\ttfamily git remote add upstream \href{https://github.com/opentoonz/opentoonz.git}{\texttt{ https\+://github.\+com/opentoonz/opentoonz.\+git}}}, additionally. 0. modify the codes.
\item {\ttfamily git checkout -\/b your-\/branch-\/name}
\begin{DoxyItemize}
\item {\ttfamily your-\/branch-\/name} is a name of your modifications, for example, {\ttfamily fix/fatal-\/bugs}, {\ttfamily feature/new-\/useful-\/gui} and so on.
\end{DoxyItemize}
\item fix codes, then test them.
\item {\ttfamily git commit} them with good commit messages. 0. {\ttfamily pull} the latest changes form the {\ttfamily master} branch of the upstream.
\item {\ttfamily git pull upstream master} or {\ttfamily git pull -\/-\/rebase upstream master}.
\item apply \href{http://clang.llvm.org/docs/ClangFormat.html}{\texttt{ clang-\/format}} with {\ttfamily toonz/sources/.clang-\/format}.
\begin{DoxyItemize}
\item {\ttfamily cd toonz/sources}
\item {\ttfamily ./beautification.sh} or {\ttfamily beautification.\+bat}.
\end{DoxyItemize}
\item {\ttfamily git commit} them.
\item {\ttfamily git push origin your-\/branch-\/name}. 0. make a pull request.
\end{DoxyItemize}\hypertarget{md__e__opentoonz__c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md3}{}\doxysection{Bugs}\label{md__e__opentoonz__c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md3}
If you find bugs, please report details about them using \href{https://github.com/opentoonz/opentoonz/issues}{\texttt{ issues}}. Please include information needed to reproduce the bug, including the operating system and information directly relating to the issue. Links to screen captures of what is observed on screen or video of specific steps to produce the problem are very helpful. ~\newline
 Then we will try to reproduce the bugs and fix them. Unfortunately, bugs can sometimes only be reproduced in your own environment, so we cannot reproduce them. If you believe you can fix the bug, please submit a pull request.\hypertarget{md__e__opentoonz__c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md4}{}\doxysection{Features}\label{md__e__opentoonz__c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md4}
If you had an idea about a new feature, please implement it and send a pull request to us. If you cannot implement the feature, please open a topic on the \href{https://groups.google.com/forum/\#!forum/opentoonz_en}{\texttt{ Google Group Page}}. It enables us to discuss implementations of the feature there. Feature requests posted on Git\+Hub without an active developer or funding will be closed to keep the issue tracker from becoming cluttered with unfulfilled feature requests.\hypertarget{md__e__opentoonz__c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md5}{}\doxysection{Translations}\label{md__e__opentoonz__c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md5}
Translation source ({\ttfamily .ts}) files for Open\+Toonz GUI are located in {\ttfamily toonz/sources/translations}. If you create new {\ttfamily .ts} files for your language or update an existing one, please send us those modifications as pull-\/requests. \href{http://doc.qt.io/qt-5.6/linguist-translators.html}{\texttt{ Qt Linguist}} is useful for translating them.

Please send us Qt message ({\ttfamily .qm}) files with {\ttfamily .ts} files if you can make the following modifications.

Open\+Toonz uses {\ttfamily .qm} files generated from {\ttfamily .ts} files. You can generate {\ttfamily .qm} files by using \href{http://doc.qt.io/qt-5.6/linguist-translators.html}{\texttt{ Qt Linguist}}. Please locate generated {\ttfamily .qm} files in {\ttfamily stuff/config/loc}. It enables the Open\+Toonz installer to install them into the {\ttfamily stuff} directory.
