Building Open\+Toonz from source requires the following dependencies\+:
\begin{DoxyItemize}
\item Git
\item GCC or Clang
\item CMake (3.\+4.\+1 or newer).
\item Qt5 (5.\+9 or newer)
\item Boost (1.\+55 or newer)
\item Lib\+PNG
\item Super\+LU
\item Lzo2
\item Free\+Type
\item Lib\+My\+Paint (1.\+3 or newer)
\item Jpeg-\/\+Turbo (1.\+4 or newer)
\item Open\+CV 3.\+2 or newer
\end{DoxyItemize}\hypertarget{md__e__opentoonz_doc_how_to_build_bsd_autotoc_md8}{}\doxysection{Open\+BSD example}\label{md__e__opentoonz_doc_how_to_build_bsd_autotoc_md8}

\begin{DoxyCode}{0}
\DoxyCodeLine{\# pkg\_add git cmake pkgconf boost qt5 lz4 usb lzo2 png jpeg glew freeglut freetype json-\/c mypaint opencv}

\end{DoxyCode}
 For programs such as Super\+LU or Jpeg-\/\+Turbo, you might need to compile from source.

Notes\+:
\begin{DoxyItemize}
\item It\textquotesingle{}s possible we also need {\ttfamily libgsl2} (Called gsl in BSD systems or maybe {\ttfamily libopenblas-\/dev} (called just blas in BSD.))
\item We may also need {\ttfamily libegl1-\/mesa-\/dev libgles2-\/mesa-\/dev libglib2.\+0-\/dev liblzma-\/dev} Install the BSD equivilency to those GNU/\+Linux packages. They might not be available as precompiled packages, so compiling from source might be required.
\item If your BSD operating system doesn\textquotesingle{}t have the up to date packages such as gsl, you may install them by compiling from source.
\end{DoxyItemize}

Continue the instructions as stated in \href{https://github.com/opentoonz/opentoonz/blob/master/doc/how_to_build_linux.md}{\texttt{ the linux guide}} to finish the compilation.
