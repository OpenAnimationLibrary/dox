\doxysubsection{Detailed Description}
The thread encapsulating a core vectorization process. 

The {\ttfamily \mbox{\hyperlink{class_vectorizer}{Vectorizer}}} class serves as a medium between Xsheets and the low-\/level vectorization methods\+: each frame of the input level is processed by {\ttfamily \mbox{\hyperlink{class_vectorizer_core}{Vectorizer\+Core}}} methods and the output is then stored on free xsheet cells. \begin{DoxyVerb}      Signals are also excanghed through this class, ensuring basic
\end{DoxyVerb}
 communications between the invoker (typically {\ttfamily \mbox{\hyperlink{class_vectorizer_popup}{Vectorizer\+Popup}}}) and the {\ttfamily \mbox{\hyperlink{class_vectorizer_core}{Vectorizer\+Core}}} class.

Since vectorization is typically a complex and resource-\/consuming task, {\ttfamily \mbox{\hyperlink{class_vectorizer}{Vectorizer}}} also inherits a {\ttfamily QThread} in order to avoid system freezes.

\begin{DoxySeeAlso}{See also}
The {\ttfamily \mbox{\hyperlink{class_vectorizer_popup}{Vectorizer\+Popup}}}, {\ttfamily \mbox{\hyperlink{class_vectorizer_configuration}{Vectorizer\+Configuration}}} and {\ttfamily \mbox{\hyperlink{class_vectorizer_core}{Vectorizer\+Core}}} classes. 
\end{DoxySeeAlso}
