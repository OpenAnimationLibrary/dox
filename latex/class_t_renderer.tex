\doxysubsection{Detailed Description}
The \mbox{\hyperlink{class_t_renderer}{TRenderer}} class provides the core operative interface to the Toonz rendering library. The \mbox{\hyperlink{class_t_renderer}{TRenderer}}\textquotesingle{}s main purpose is that of invoking a render process on a group of separate threads through the {\bfseries{\mbox{\hyperlink{class_t_renderer_a478735dc2935569993603911e116e0f5}{start\+Rendering()}}}} methods. After this is done, the activated process communicates with any \mbox{\hyperlink{class_t_render_port}{TRender\+Port}} previously set through the {\bfseries{add\+Port()}} method in order to notify the render status. The rendering process can be interrupted with the stop\+Rendering() method, which makes all rendering threads quit safely at the most appropriate time; the more specific abort\+Rendering() method may be used to terminate a render process id in case more than one process is running. ~\newline
 ~\newline
Technically, this class is an {\itshape interface} to the actual rendering class, which remains in background. This means that users have no necessity of maintaining the \mbox{\hyperlink{class_t_renderer}{TRenderer}} instance alive as a render process is running. We may, for example, create a \mbox{\hyperlink{class_t_renderer}{TRenderer}}, set its properties, launch the render, and release the \mbox{\hyperlink{class_t_renderer}{TRenderer}} instance immediately after as the internal rendering class will continue working on. TRenderers may therefore be copied and assigned freely to refer an already existing internal rendering instance. These are created anew only using the default \mbox{\hyperlink{class_t_renderer}{TRenderer(int)}} constructor. ~\newline
 ~\newline
On the other hand, \mbox{\hyperlink{class_t_renderer}{TRenderer}}\textquotesingle{}s lifespan may last longer than that of a single rendering process. This is especially useful when it is known that multiple rendering processes share some common resources (likewise, intermediate cache results), which can be maintained in the same internal \mbox{\hyperlink{class_t_renderer}{TRenderer}} representation for later use. ~\newline
 ~\newline
Some final notes about methods dealing with Toonz\textquotesingle{}s lower-\/level API. Direct invocation of rendering functions, such as the TRaster\+Fx\+::compute() method, is discouraged. Further improvements of the \mbox{\hyperlink{class_t_renderer}{TRenderer}}\textquotesingle{}s API will eventually make the need for these calls obsolete. However, we\textquotesingle{}ve implemented the possibility to rely on \mbox{\hyperlink{class_t_renderer}{TRenderer}}\textquotesingle{}s management support even in those cases -\/ it can be done by declaring the render process\textquotesingle{} interesting events through the apposite  declare..() \texorpdfstring{$<$}{<}{\itshape } \texorpdfstring{$>$}{>} methods and invoking \mbox{\hyperlink{class_t_renderer_ad78d6afa9d762067e813a9b745d62690}{install()}} and \mbox{\hyperlink{class_t_renderer_a7e3e6555330313d89f60aa62c52cde93}{uninstall()}} on each rendering thread you\textquotesingle{}ll raise for the process.
