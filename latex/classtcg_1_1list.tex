\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename T$>$\newline
class tcg\+::list$<$ T $>$}
Double linked list container class with indexed access to elements. 

This class is the result of implementing a double linked list using a dynamic random access sequential container as the underlying pool for the list\textquotesingle{}s nodes.

This is somewhat different than supplying a custom allocator to an existing list implementation, like {\ttfamily std\+::list}, since\+:


\begin{DoxyItemize}
\item No memory model of the underlying container is actually required. 
\item The underlying container provides explicit access to nodes using their associated {\itshape insertion indexes}, so {\ttfamily std\+::list} could not be used as-\/is. 
\end{DoxyItemize}~\newline
 The nodes pool (see {\ttfamily \mbox{\hyperlink{classtcg_1_1list__base}{list\+\_\+base}}}) is an internal {\itshape lazy} container that saves released nodes {\itshape without} actually deallocating them (of course, their content is properly destroyed). They are rather stored into an internal {\itshape stack} of released nodes, ready to be reacquired. \begin{DoxyParagraph}{Features}
\begin{DoxyItemize}
\item Insertion indexes are automatically generated keys with the same access time of the underlying container (typically super-\/fast {\ttfamily O(1)}). \item Stable indexes and iterators with respect to {\itshape every operation}. \item Copying an indexed list {\itshape preserves} the original indexes. \item Indexes are {\itshape process independent} -\/ they can be saved on file and restored later (although you probably want to {\itshape squeeze} the list before saving).\end{DoxyItemize}

\end{DoxyParagraph}
\begin{DoxyParagraph}{Use cases}
This class is surprisingly useful for {\itshape cross-\/referencing}. A graph data structure, for example, can be implemented using a pair of indexed lists for vertexes and edges.
\end{DoxyParagraph}
\begin{DoxyParagraph}{Known issues}
The lazy nature of this container class may require {\itshape squeezing} the list\textquotesingle{}s content at strategic places in your code, as noted before. Beware that composite cross-\/referencing may be hard to track {\itshape precisely} during this particular operation.
\end{DoxyParagraph}
\begin{DoxyRemark}{Remarks}
Current implementation explicitly uses an {\ttfamily std\+::vector} class as the underlying random access container. Future improvements will make the container class a template parameter.
\end{DoxyRemark}

\end{DoxySeeAlso}
