Detailed Description

This file contains the description of a C++ idiom that can be used to perform controlled access on an object.

Controlled access forces users to invoke a function every time an object is accessed, at compile-time.

Some notable cases for this idiom include the "frequent notifier / lazy update" scenario, where an object is frequently notified for status updates, but many updates can be spared until the data is finally accessed.
