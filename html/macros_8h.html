The TCG_ASSERT instruction is an assertion that also provides a valid failure statement in case the asserted expression fails.

Assert statements are used to enforce pre- and post-conditions to algorithms, in a way that is not intended to be broken - in other words, after a failed assert statement we should expect nothing more than undefined behavior. The assert statement itself will signal a failed expression, and then will force the program to quit <\I>.

However, there exist cases in which at the same time:

  • It is known that a pre/post-condition could be possibly broken
  • It should still be considered an error
  • Yet, a worst-case bail-out policy must be provided

When this happen, you should:

  • Feel offended - it should really not<\B> happen. Errors in meeting requirements should never be tolerated
  • Use TCG_ASSERT. It does not solve things, but is a simple statement and defines a trackable (grep-able) string in the project
