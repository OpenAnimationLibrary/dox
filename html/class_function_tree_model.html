Detailed Description

The Function Editor's (tree-like) model, as in the model-view architecture.

This class represents the data associated to Toonz's Function Editor panels in a view-independent way. The model's purpose is that of representing all channels of the objects in a scene. A channel is here intended as an animatable parameter represented by a single real-valued function.

Animatable objects are currently subdivided in two main types: stage objects (which consist roughly all the objects represented in the stage schematic view, including cameras, spline curves and pegbars), and fxs. Stage objects typically feature a uniform channels group structure, whereas each fx type have a different set of parameters (and thus channels). Recently, column objects can sport an additional group of channels, related to Plastic skeleton animations (see TnzExt library).
