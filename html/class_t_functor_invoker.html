Detailed Description

The TFunctorInvoker is a singleton class that can be used to invoke functions as Qt slots without having to inherit from QObject.

Rationale
Signal and slots are an important part of the Qt framework - however, using them requires to explicitly inherit from QObject. This singleton object is provided to avoid the memory overhead of inheriting from QObject in the case of simple function calls - notably functions that need to synchronize with Qt's main event loop.
Usage
Inherit from TFunctorInvoker::BaseFunctor and allocate a new instance on the heap, to be supplied to the TFunctorInvoker::invoke() slot.
The following code exemplifies a correct usage for this class:
class MyFunctor final : public TFunctorInvoker::BaseFunctor
{
MyParams m_params; // Function
parameters
public:
MyFunctor(const MyParams& params) : m_params(params) {}
void operator()()
{
// The function body, operating on m_params
}
};
//------------------------------------------------------------------
void invokeMyFunctor_queued(const MyParams& params)
{
QMetaObject::invokeMethod(TFunctorInvoker::instance(), "invoke",
Qt::QueuedConnection,
Q_ARG(void*, new MyFunctor(params))
);
}
