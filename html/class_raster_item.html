Detailed Description

The RasterItem class represents a reusable output raster for rendering purposes. RasterItems are used as TRenderer's output rasters in order to avoid the overhead of allocating one raster for each rendered frame. Each frame-rendering task will lock a RasterItem as preallocated output before starting the render, therefore changing the RasterItem::m_busy attribute accordingly. As each frame is rendered on a separate thread, the number of RasterItems that TRenderer will allocate depends on the number of rendering threads specified to it.
