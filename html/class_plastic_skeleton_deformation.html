Detailed Description

PlasticSkeletonDeformation models the deformation of a group of PlasticSkeleton instances.

Description

A PlasticSkeleton instance is typically used to act as a deformable object - in other words, it defines the 'original' form of a hierarchy of vertices that can be manipulated interactively to obtain a 'deformed' configuration.

PlasticSkeletonDeformation represents a deformation of PlasticSkeleton objects, therefore acting primarily as a collection of PlasticSkeletonVertexDeformation instances - one per skeleton vertex. The collection is an associative container mapping a vertex name <\I> to its deformation; using names as keys is a useful abstraction that allows vertex deformations data (eg keyframes) to be copied to skeleton deformations whose skeletons have a different internal configuration (ie vertex indices and such).

Each vertex deformation also stores a unique hook number <\I> that can be used during xsheet animation to link stage objects to a skeleton vertex.

The Skeletons group

The PlasticSkeletonDeformation implementation has been extended to work on multiple skeletons for one class instance. This class stores a map associating skeleton indices to the skeleton instances, that can be used to select a skeleton to be deformed with the deformation's data.

Vertices in different skeleton instances share the same animation if their name is the same.

This class provides an animatable parameter that is intended to choose the active skeleton along an xsheet timeline. It is retrievable through the skeletonIdsParam() method.

Notable implementation details

In current implementation, a PlasticSkeletonDeformation keeps shared ownership of the skeletons it is attached to. It is therefore intended to be a container of said skeletons.
